# P32-T1 - Review Bundle Format

## Phase

[Phase 32 - Collaboration and Review Handoff](../../phases/phase-32-collaboration-review-handoff.md)

## Objective

Define review bundle manifest and content structure for sharing Ralph run context outside the live desktop session.

## Deliverables

- Define manifest fields for project, run, blueprint, specs, artifacts, evidence, decisions, diffs, delivery, and limitations.
- Define directory structure for markdown summaries, JSON metadata, screenshots, logs, and attachments.
- Define versioning and compatibility metadata.
- Define required redaction and exclusion rules.
- Document which bundle sections are optional or role-specific.

## Dependencies

- Delivery, artifact, visual evidence, and policy data sources are available.

## Acceptance Criteria

- Review bundle format is documented and versioned.
- The format can represent completed and in-progress runs.
- Secret and excluded file rules are part of the format.
- Future import logic can validate bundle compatibility.
