# P5-T3 - Implement Update Flow

## Phase

[Phase 05 - Reliability and Distribution](../../phases/phase-05-reliability-and-distribution.md)

## Objective

Add version visibility and update handling so Knuthflow can improve after installation without forcing manual reinstall every time.

## Deliverables

- Current app version visible in the UI
- Update check and notification behavior
- Apply-update flow or controlled handoff to installer behavior
- Release notes surface if appropriate

## Dependencies

- [P5-T2](./p5-t2-define-packaging-and-signing.md)

## Acceptance Criteria

- Users can tell what version they are running
- Available updates are surfaced clearly
- The update path does not corrupt local app data
- Update behavior matches the chosen distribution channel

Not Implemented
