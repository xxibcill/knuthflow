# P5-T4 - Create QA Matrix and Release Checklist

## Phase

[Phase 05 - Reliability and Distribution](../../phases/phase-05-reliability-and-distribution.md)

## Objective

Define the minimum test matrix and release checklist required to ship the wrapper with confidence.

## Deliverables

- Smoke test list for core user flows
- Platform coverage matrix
- Regression checklist for PTY, settings, sessions, and packaging
- Release signoff checklist

## Dependencies

- [P5-T1](./p5-t1-add-process-supervision-and-recovery.md)
- [P5-T2](./p5-t2-define-packaging-and-signing.md)
- [P5-T3](./p5-t3-implement-update-flow.md)

## Acceptance Criteria

- The release bar is written down
- Core flows have explicit verification steps
- Platform-specific risk areas are covered
- Shipping decisions no longer rely on memory alone
