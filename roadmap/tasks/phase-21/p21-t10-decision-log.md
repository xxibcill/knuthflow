# P21-T10 - Decision Log

## Phase

[Phase 21 - Ralph Product Source of Truth](../../phases/phase-21-ralph-product-source-of-truth.md)

## Objective

Record the strategic decisions behind the Ralph product refocus, compatibility policy, and retained runtime foundations.

## Deliverables

- Add a decision-log entry explaining the move from Knuthflow wrapper positioning to Ralph operator desktop positioning.
- Document why terminal, PTY, workspace, editor, settings, and history capabilities are retained as support systems.
- Document the intended compatibility approach for visible brand names, package identity, preload API names, database names, and user data paths.
- Document the decision to introduce Ralph-first UX before any broad internal rename.
- Document known tradeoffs, including possible temporary inconsistency between visible branding and internal identifiers.
- Add open questions for unresolved brand, package, data migration, or release identity choices.

## Dependencies

- Phase 21 terminology and Phase 22 brand policy discussions are underway.
- Any unresolved choice is acceptable if it is explicitly marked as open.

## Acceptance Criteria

- `PRD.md` includes decision-log entries for the Ralph refocus and compatibility stance.
- The decision log gives future contributors enough context to avoid repeating the same debate.
- Open questions are explicit, assigned if possible, and tied to later phases.
- No compatibility-sensitive decision is left implicit.
