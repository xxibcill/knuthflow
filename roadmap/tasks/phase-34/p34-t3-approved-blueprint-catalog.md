# P34-T3 - Approved Blueprint Catalog

## Phase

[Phase 34 - Enterprise Workspace Governance](../../phases/phase-34-enterprise-workspace-governance.md)

## Objective

Add approved blueprint catalog and version requirements for governed Ralph work.

## Deliverables

- Define catalog format for approved blueprint ids, versions, categories, and allowed customization.
- Show approved blueprints during intake and blueprint selection.
- Warn or block unapproved blueprints based on team policy.
- Track blueprint version used by each project.
- Provide update guidance when a project uses an outdated approved blueprint.

## Dependencies

- Blueprint system and team profile model exist.

## Acceptance Criteria

- Team-managed projects can require approved blueprints.
- Operators see why a blueprint is approved, outdated, or blocked.
- Blueprint version usage is recorded.
- Solo projects can still use non-catalog blueprints when no governance applies.
