# P20-T4 - Blueprint Intake Integration

## Phase

[Phase 20 - Skill Library and Blueprint System](../../phases/phase-20.md)

## Objective

Integrate blueprint selection into the app intake flow as an alternative to freeform brief entry.

## Deliverables

- Intake screen with two paths: "Start from blueprint" or "Describe your app"
- Blueprint picker: searchable grid of blueprints with preview
- Blueprint preview: shows what the app will include (starter template, specs, tasks)
- Selected blueprint pre-fills intake fields: stack, platform targets, app type
- Operator can customize pre-filled values before bootstrap

## Dependencies

- P20-T2 (Blueprint UI) complete

## Acceptance Criteria

- Intake offers blueprint selection alongside freeform brief
- Blueprint selection pre-fills intake fields
- Operator can override pre-filled values
- Bootstrap uses selected blueprint's starter template and specs
