# P4-T2 - Build Settings UI

## Phase

[Phase 04 - Settings and Local State](../../phases/phase-04-settings-and-local-state.md)

## Objective

Expose key wrapper settings in the UI so users do not need to configure Knuthflow through hidden files or environment tweaks.

## Deliverables

- Settings screens or panels
- Editable CLI path, default workspace, and launch preference controls
- Safety-related controls aligned with the app model
- Save, cancel, and validation feedback behavior

## Dependencies

- [P4-T1](./p4-t1-persist-settings-and-profiles.md)

## Acceptance Criteria

- Users can inspect and edit core app settings in the GUI
- Invalid settings surface actionable validation
- Saved settings are reflected in launch behavior
- Settings screens remain understandable without documentation
