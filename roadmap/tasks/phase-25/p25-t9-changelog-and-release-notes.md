# P25-T9 - Changelog And Release Notes

## Phase

[Phase 25 - Ralph Release Readiness](../../phases/phase-25-ralph-release-readiness.md)

## Objective

Write release notes explaining the Ralph refocus, user-visible changes, compatibility policy, data migration behavior, and known limitations.

## Deliverables

- Add changelog entry for the Ralph-focused release.
- Summarize user-visible changes to branding, navigation, first screen, project flow, API names, and data compatibility.
- Document whether existing Knuthflow-branded data or aliases still work.
- Document known limitations and non-goals.
- Document upgrade notes for operators and contributors.
- Keep release notes concise enough for users while linking to deeper docs for details.

## Dependencies

- Phase 21-24 changes are complete enough to describe accurately.
- Compatibility and packaging verification results are known.

## Acceptance Criteria

- Release notes do not overpromise unsupported Ralph autonomy.
- Compatibility and data behavior are explicit.
- Known limitations are honest and actionable.
- Changelog references the Ralph-focused release coherently.
